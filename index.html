Mi proceso de trabajo

1.- Análisis de usuario y benchmark

Un buen proyecto comienza haciendo un análisis de usuario para definir el público objetivo y la identidad de la marca (Solo si esto no está definido antes). Luego se hace un análisis comparativo de distintos referentes, para saber qué hacen bien y qué cosas están demás.

2.- Brainstorming y propuestas de diseño

Una vez realizados los análisis, hago una lluvia de ideas para poder realizar múltiples propuestas de diseño y otorgar múltiples soluciones. Teniendo las ideas solidificadas, realizo propuestas de diseño al cliente para llegar a un acuerdo para decidir cuál es la que le parece mejor.

3.- Prototipado del diseño

Para entregar una mejor idea de cómo será el sitio al final, se realiza un prototipo del diseño propuesto, con funcionalidades y efectos visuales en este. Se realizan testeos de funcionalidad con múltiples usuarios para luego corregir posibles errores o fallas en el diseño.

4.- Programación del sitio

Luego de ser testeado el prototipo, se comienza con la programación del sitio, replicando directamente la maqueta utilizada.

5.- Testeo y correcciones finales

Al terminar de programar, se sube el sitio web al hosting correspondiente y se asocia al dominio solicitado. Luego se prueba el sitio web estando en linea para posteriormente realizar las correcciones finales.

6.- Entrega

Cuando se comprueba que el funcionamiento del sitio es el correcto, se entrega el sitio web al cliente, junto con el hosting y dominio.

Axit PSD

Axit PSD es una plantilla gratuita para landing pages hecha en Photoshop. Se tuvo la tarea de replicarla utilizando solamente HTML y CSS, sin responsive.

Luego de hacerlo como tarea, decidí tomar la plantilla nuevamente para codificarla desde 0, utilizando BootStrap y media queries para el implemento de responsive y JavaScript para pequeñas interacciones que tiene la página.

bstm

Este portafolio también está incluido dentro del mismo

A diferencia de otros proyectos en los cuales no se tuvo mucho tiempo para organizarse o quizá se tuvo muy poco tiempo para organizar y trabajar, en bstm el proceso fue largo. Se inició por una organización de tareas por realizar, se pasó por múltiples tomas de decisiones y momentos en los que se comenzó de 0 y finalmente pasó por un proceso de wireframing (maquetado sin colores) y mockup. Una vez se hizo todo esto, se pasó a la ejecución utilizando HTML, CSS y JS, a pesar de que se tenía pensado trabajar en WordPress en un principio.

El resultado del largo tiempo que tomó el proyecto fue una página web simple con interacciones pequeñas, manteniendo una estética con colores cálidos y tipografías sin serifas, modernas y agradables a la vista.

Meraki Food

Meraki food es un emprendimiento dedicado a entregar servicios de banquetería y coctelería a eventos grantes, principalmente a eventos empresariales o similares.

Durante mi práctica profesional se me asignó la tarea de rediseñar el sitio que ya estaba hecho, por lo que tuve la oportunidad de rehacer la estructura, navegación y estética del sitio a mi manera.

Trabajé la maqueta utilizando figma, donde elegí la estética general del sitio y cómo iba a estar estructurado. Utilicé los colores del logo del emprendimiento con unas ligeras modificaciones, a la vez que elegí una tipografía sin serifas moderna y una serif para los textos que tuviera un tono un poco más formal debido a que el emprendimiento está dedicado a empresas grandes.

Al final, se pasó todo lo de la maqueta a través de Elementor. Lamentablemente no se pudieron hacer grandes cambios a la página de los productos ya que me asignaron otras tareas. El resultado fue conseguido en alrededor de una semana.
